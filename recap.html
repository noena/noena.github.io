Lesson Recap:

Ruby-wise

  • if we see the if ... return true else return false, we don't need it
  • the last expression in a function will be what is returned by it
  • indenting helps to identify sub-expressions in expressions
  • function/method/variable names should be meaningful
  • Avoid puts in functions, unless debugging
  • Remember puts is not the same as return
  • return will stop function execution
  • Tests help, if you have enough variety of test cases
  • Writing code into functions/methods helps us test it
  • Functions/methods can be written to take in default parameters i.e.
    •         def function_name(param_name = default_parameter)
        # do stuff
        end
  • in general, prefer one line to do one thing in the code. But use your discretion
  • method names with "!" (pronounced 'bang') are more dangerous
  • sort does not mutate the original array, instead returning a new array. But sort! mutates and returns the original array
  • Array math operations are similar to set operations, like intersection '&' (pronounced 'ampersand') and union '|' (pronounced 'pipe')
  • String characters can be sorted in an array, because each character corresponds to an ascii code number
  • Hashes are ruby objects that store key value pairs. Retrieval of values is like in an array, except you put the key in the square brackets i.e. hash['a'].
  • Hash values can be mutated by assigning to a key using the square brackets, i.e. hash[] =
  • Using Hash.new() you can set a default variable for missing keys. e.g.
    •         hash = Hash.new(0) #=> {}
        hash['a']          #=> 0

        hash = {}          #=> {}
        hash['a']          #=> nil
  • String#chars gets you an array of all the characters in the string. More efficient than String#split
  • Regular Expressions, are patterns that are used to match strings. In Ruby, regular expression objects belong to the RegExp class
  • array.map(&:negative?) is equivalent to array.map { |element| element.send(:negative?) }. What is happening is that the &:negative in the params translates into :negative?.to_proc, which returns a proc (which is like a block), that functions like { |x| x.send(:negative?) }.
